========================================
 Sidekey 电脑端 · macOS 使用说明
========================================

【安装】
1. 双击下载到的 Sidekey.dmg。
2. 把窗口里的 Sidekey 图标拖到『应用程序』文件夹。
3. 到『应用程序』里双击 Sidekey 启动。

   · 如果提示"无法打开, 因为来自身份不明的开发者"(只有未签名的测试版会这样):
     右键(或按住 Control 点)Sidekey → 选『打开』→ 再点『打开』。只需这一次。
     (正式签名+公证过的版本直接双击即可, 不会有这个提示。)

========================================
 第一次启动会遇到的三件事 (很重要)
========================================
1. 辅助功能授权
   会弹出"Sidekey 想要使用辅助功能控制这台电脑"。
   点『打开系统设置』→ 在『隐私与安全性 > 辅助功能』里把 Sidekey 开关打开。
   然后【退出 Sidekey, 再重新打开】(授权后必须重启一次才生效)。
   没有这一步, 手机发来的按键在电脑这边不会有任何反应。

2. 本地网络
   可能会问"Sidekey 想查找并连接本地网络上的设备", 点『允许』, 否则手机连不上。

3. 二维码
   启动后会自动弹出一张二维码图片(用『预览』打开)。
   手机打开 Sidekey App → 点左上角电脑名 → 『扫码添加电脑』, 扫这张二维码即可。
   (二维码没扫上: 重新打开那张图, 或在 App 里用"手动添加"。)

========================================
 怎么用 / 怎么停
========================================
· 手机和这台 Mac 连同一个 Wi-Fi。
· Sidekey 启动后, 图标会停在程序坞(Dock)里 —— 它就是服务端, 用的时候一直开着。
· 停止: 右键 Dock 里的 Sidekey 图标 → 『退出』。

----------------------------------------
 给开发者: 怎么自己打包 / 签名 / 公证
----------------------------------------
A. 打包 (任何装了 Python3 的 Mac):
   双击  build_app.command  →  生成 dist/Sidekey.app (未签名, 自己用够了)。
   注: 打出来的 .app 是本机架构 (Apple Silicon = arm64)。要兼容老的 Intel Mac
       得做 universal2 构建, 以后再说。

B. 签名 + 公证 + 打 dmg (要发给别人 / 放下载页时):
   一次性准备:
     1) 加入付费 Apple Developer Program 后, 到 developer.apple.com → Certificates
        申请并下载 "Developer ID Application" 证书, 双击导入钥匙串。
     2) 终端跑一次 (把公证凭据存进钥匙串, 之后发版不用再输密码):
          xcrun notarytool store-credentials sidekey-notary \
            --apple-id 你的AppleID邮箱 --team-id 你的TEAMID --password App专用密码
        (App 专用密码: appleid.apple.com → 登录与安全 → App 专用密码 里生成)
   之后每次发版:
     双击  sign_notarize_dmg.command  →  生成 dist/Sidekey.dmg (已签名+公证)。
